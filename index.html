SQL: Your Daily Driver

Why SQL Matters

  • Foundation for everything you build
  • Interface between applications
  • Get data in, store data out
  • Answer most questions with GROUP, SUM, AVG

Example

-- Player distance across sessions
SELECT 
    player_id,
    SUM(total_distance) as total,
    COUNT(DISTINCT session_id)
FROM player_drills
WHERE player_id = 'PLAYER_123'
GROUP BY player_id;

Statistics You Actually Use

Catching Bad Data: Heart Rate

  • GPS device loses contact when improperly fitted and skew session averages
  • Compare average HR to expected range for that training load
  • Flag sessions where avg HR is unusually high for similar effort
  • Understanding "normal" vs "outlier" catches the problem

Stats aren't theoretical. Understanding distributions and standard deviation is how you protect data quality daily.

The Reality: Working with APIs

Data Format & Validation

API returns deeply nested JSON. Player positions, timestamps, and metadata all mixed together.

The API lets you filter for validated games only, which helps prevent faulty data from entering your system.

Delays & Incomplete Data

Data doesn't always arrive on time. Sometimes it's incomplete.

Stakeholders are waiting, so you need monitoring and alerts to catch issues early.

Your job: Transform messy API responses into clean, reliable data analysts can trust
